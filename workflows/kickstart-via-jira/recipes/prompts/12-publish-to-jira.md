# Publish to Jira

This workflow creates or updates a DOTBOT tracking issue under the parent initiative in Jira and posts research summaries as comments. It uses the Atlassian MCP tools directly (no separate credentials needed).

## Prerequisites

- `briefing/jira-context.md` must exist (for the parent Jira key and cloud ID)
- Atlassian MCP server must be available
- Research artifacts should exist (but workflow handles missing files gracefully)

## Your Task

### Step 1: Read Context

```
Read({ file_path: ".bot/workspace/product/briefing/jira-context.md" })
```

Extract:
- **Jira Key** from the metadata table (e.g., `PROJ-1234`)
- **Atlassian Cloud ID** from the Organisation Settings table

If the Cloud ID is not in jira-context.md, discover it:
```
mcp__atlassian__getAccessibleAtlassianResources()
```

### Step 2: Search for Existing DOTBOT Issue

Search for an existing DOTBOT tracking issue:
```
mcp__atlassian__searchJiraIssuesUsingJql({
  cloudId: "{CLOUD_ID}",
  jql: "parent = {JIRA_KEY} AND summary ~ \"DOTBOT\"",
  limit: 5
})
```

If found, use its key for Step 4. If not found, proceed to Step 3.

### Step 3: Create DOTBOT Tracking Issue

Create a child task under the parent initiative:
```
mcp__atlassian__createJiraIssue({
  cloudId: "{CLOUD_ID}",
  projectKey: "{PROJECT_KEY}",
  issueTypeName: "Task",
  summary: "{JIRA_KEY}--DOTBOT Research Artifacts",
  description: "Auto-generated by dotbot. Contains research artifacts and analysis for this initiative.",
  parent: "{JIRA_KEY}"
})
```

Record the new issue key.

### Step 4: Read and Post Research Artifacts

Read each artifact that exists (skip missing ones gracefully):

```
Read({ file_path: ".bot/workspace/product/mission.md" })
Read({ file_path: ".bot/workspace/product/roadmap-overview.md" })
Read({ file_path: ".bot/workspace/product/research-summary.md" })
Read({ file_path: ".bot/workspace/product/research-internet.md" })
Read({ file_path: ".bot/workspace/product/research-documents.md" })
Read({ file_path: ".bot/workspace/product/research-repos.md" })
```

Also check for deep-dive summaries:
```
Glob({ pattern: "*.md", path: ".bot/workspace/product/briefing/repos" })
```

For each existing artifact, post a comment on the DOTBOT issue. Group related artifacts into a single comment to avoid flooding. Suggested grouping:

**Comment 1 — Mission & Roadmap:**
```
mcp__atlassian__addCommentToJiraIssue({
  cloudId: "{CLOUD_ID}",
  issueIdOrKey: "{DOTBOT_KEY}",
  body: "h2. Mission Statement\n\n{mission.md content excerpt — first 500 lines}\n\nh2. Roadmap Overview\n\n{roadmap-overview.md content excerpt}"
})
```

**Comment 2 — Research Summary:**
```
mcp__atlassian__addCommentToJiraIssue({
  cloudId: "{CLOUD_ID}",
  issueIdOrKey: "{DOTBOT_KEY}",
  body: "h2. Research Summary\n\n{research-summary.md content}"
})
```

**Comment 3 — Detailed Research (if content is substantial):**
```
mcp__atlassian__addCommentToJiraIssue({
  cloudId: "{CLOUD_ID}",
  issueIdOrKey: "{DOTBOT_KEY}",
  body: "h2. Internet Research\n\n{excerpt}\n\nh2. Document Research\n\n{excerpt}\n\nh2. Repository Research\n\n{excerpt}"
})
```

**Comment 4 — Deep Dives (if any exist):**
Post a summary of deep dive findings with key metrics from each repo.

### Step 5: Report Results

Report:
- Whether the DOTBOT issue was created or already existed
- The issue key and URL
- How many artifacts were posted
- Any artifacts that were missing or failed

## Critical Rules

- Use Atlassian MCP tools (`mcp__atlassian__*`) for all Jira operations — do NOT use REST API calls or the `jira_publish` tool
- The parent key must come from `jira-context.md`, not hardcoded
- Do NOT modify any research artifacts — this is a read-only publish step
- If the DOTBOT issue already exists, do not create a duplicate — add comments to the existing one
- Keep comment content concise — use excerpts (first ~500 lines) for large artifacts
- Only post files that actually exist — skip missing optional artifacts gracefully
- Use Jira wiki markup in comments (h2., *, |table|format|, {code}, etc.)
